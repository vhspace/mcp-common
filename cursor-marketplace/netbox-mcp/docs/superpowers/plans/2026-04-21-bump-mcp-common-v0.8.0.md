# Bump mcp-common to v0.8.0 — Structured Logging & Timing Telemetry Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump `mcp-common` from the pinned pre-tag rev (`d1d340f`, pre-v0.7.0) to tagged release `v0.8.0` in netbox-mcp, wire up the new structured logging pipeline and timing telemetry across the MCP server and CLI, and add a smoke test validating the logging stack.

**Architecture:** Small, additive, mostly kwarg-level changes. The module-level `logger` in `server.py:29` is reused; `cli.py` gains one. `setup_logging` is called with `name="netbox-mcp"` and `system_log=True` so logs route to journald on Linux. The existing `install_cli_exception_handler` call and the five `@mcp_remediation_wrapper` decorators each get a `logger=` kwarg so remediation events are traced. A new `tests/test_logging_smoke.py` exercises `setup_logging`, `JSONFormatter`, and `timed_operation` directly against mcp-common.

**Tech Stack:** Python 3.12+, `uv` for dependency management, pytest for tests, `mcp-common==0.8.0` (git tag), ruff/mypy for linting.

**Tracking:** Implements [vhspace/netbox-mcp#36](https://github.com/vhspace/netbox-mcp/issues/36) — the issue targets v0.7.0, but we use v0.8.0 (current latest; bundles the v0.7.x logging/telemetry features plus the v0.8.0 slim-`ToolError` breaking change from [mcp-common#39](https://github.com/vhspace/mcp-common/pull/39)). `compute_http_error_fingerprint` and `poll_with_progress` are both unused in netbox-mcp. Item 3 of the issue is addressed via `timed_operation` in the logging smoke test and as an available primitive for future use. The v0.8.0 slim-`ToolError` behavior is verified non-breaking against the current netbox-mcp test suite: `tests/test_error_handling.py::test_get_objects_wraps_http_errors` (`match="HTTP 404"`) and `tests/test_search.py::test_invalid_object_type_raises_error` (`match="Invalid object_type"`) both still pass because their substring matches survive the new two-line shape.

**Baseline:** `uv run pytest -q` → 207 passed, 1 pre-existing failure in `tests/test_cli_search.py::TestHelpText::test_lookup_help_text` (help-text assertion unrelated to this work). All other tests must stay green.

**v0.8.0 tag (for `[tool.uv.sources]`):** pinned via `tag = "v0.8.0"` (commit `2f5993694c939ee33dfe84644521b8ec94c3b5fa`).

## Task 2: Write failing logging smoke test

**Files:**
- Create: `tests/test_logging_smoke.py`

This test validates the mcp-common logging contract. Writing it first makes the v0.8.0 API surface explicit and guards future bumps.

- [ ] **Step 1: Create the test file**

Create `tests/test_logging_smoke.py` with this content:

```python
"""Smoke test for the mcp-common logging + telemetry stack.

Validates that the features netbox-mcp relies on (per issue #36) work as
advertised: setup_logging wiring, JSONFormatter output fields, and
timed_operation event emission.
"""

from __future__ import annotations

import io
import json
import logging
import time

import pytest

from mcp_common.logging import JSONFormatter, setup_logging, timed_operation


def test_setup_logging_accepts_system_log_and_name() -> None:
    """setup_logging(name=..., system_log=True) must return a configured Logger.

    This is the exact call shape netbox-mcp uses in server.py's _initialize().
    """
    logger = setup_logging(
        name="netbox-mcp-smoke",
        level="INFO",
        json_output=False,
        system_log=True,
    )
    assert isinstance(logger, logging.Logger)


def test_json_formatter_emits_required_fields() -> None:
    """JSONFormatter output must contain timestamp, level, logger, message."""
    buf = io.StringIO()
    handler = logging.StreamHandler(buf)
    handler.setFormatter(JSONFormatter())

    test_logger = logging.getLogger("smoke.json")
    test_logger.handlers.clear()
    test_logger.addHandler(handler)
    test_logger.setLevel(logging.INFO)
    test_logger.propagate = False

    test_logger.info("hello")
    handler.flush()

    record = json.loads(buf.getvalue().strip())
    assert record["level"] == "INFO"
    assert record["logger"] == "smoke.json"
    assert record["message"] == "hello"
    assert "timestamp" in record


def test_timed_operation_emits_timing_fields() -> None:
    """timed_operation must emit a JSON event with operation, ok, actual_s."""
    buf = io.StringIO()
    handler = logging.StreamHandler(buf)
    handler.setFormatter(JSONFormatter())

    test_logger = logging.getLogger("smoke.timing")
    test_logger.handlers.clear()
    test_logger.addHandler(handler)
    test_logger.setLevel(logging.INFO)
    test_logger.propagate = False

    with timed_operation(test_logger, "smoke-op", expected_s=1.0):
        time.sleep(0.01)
    handler.flush()

    lines = [ln for ln in buf.getvalue().splitlines() if ln.strip()]
    assert lines, "timed_operation produced no log output"
    event = json.loads(lines[-1])
    assert event["operation"] == "smoke-op"
    assert event["ok"] is True
    assert isinstance(event["actual_s"], (int, float))
    assert event["actual_s"] >= 0.0


def test_timed_operation_marks_failure_on_exception() -> None:
    """timed_operation must set ok=False when the wrapped block raises."""
    buf = io.StringIO()
    handler = logging.StreamHandler(buf)
    handler.setFormatter(JSONFormatter())

    test_logger = logging.getLogger("smoke.timing.fail")
    test_logger.handlers.clear()
    test_logger.addHandler(handler)
    test_logger.setLevel(logging.INFO)
    test_logger.propagate = False

    with pytest.raises(RuntimeError, match="boom"):
        with timed_operation(test_logger, "smoke-op-fail"):
            raise RuntimeError("boom")
    handler.flush()

    lines = [ln for ln in buf.getvalue().splitlines() if ln.strip()]
    assert lines, "timed_operation produced no log output on failure"
    event = json.loads(lines[-1])
    assert event["operation"] == "smoke-op-fail"
    assert event["ok"] is False
```
